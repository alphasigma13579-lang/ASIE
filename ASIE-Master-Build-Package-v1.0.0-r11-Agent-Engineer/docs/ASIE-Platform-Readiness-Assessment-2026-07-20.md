# تقييم جاهزية منصة ASIE — 20 يوليو 2026

## خلاصة القرار

المنصة ناضجة معمارياً في طبقة الحقيقة المحلية، Snapshot، الأدلة، التشغيل، والاختبارات. لكنها ليست جاهزة بعد كمنتج SaaS خارجي كامل؛ لأن جزءاً من تجربة السوق والخرائط والبيانات الخارجية ما زال تجريبياً ومحلياً، كما أن Profiles الجهات التمويلية مرجعية وليست سياسات رسمية.

## خط الأساس المتحقق

- `pnpm build`: ناجح.
- اختبارات Python: 153 ناجحة، اختبار واحد متخطى لبيئة تصدير DOCX.
- AAS Runtime مجمد ولا توجد حاجة لتغييره في هذه المرحلة.
- مسار التقرير التمويلي موجود: HTML، DOCX، XLSX، PDF، الجاهزية، المراجعة، والإصدار المحلي.

## ما هو قوي

1. مصدر الحقيقة واضح: التقرير إسقاط من Snapshot ولا يعيد الحساب من الواجهة.
2. وجود Evidence Ledger وTransformation Lineage يتيح تتبع الرقم والافتراض.
3. وجود Review Overlay وسجل إصدار محلي يمنع تسليم نسخة غير مراجعة.
4. وجود تقارير متعددة الصيغ من نفس الإسقاط يقلل اختلاف الأرقام بين PDF وDOCX وXLSX.
5. الاختبارات الحالية واسعة نسبياً، والبناء الأمامي ناجح.
6. الحدود الأمنية واضحة: الشبكة والمزودون الخارجيون والقبول التمويلي غير مفعلة تلقائياً.
7. تجربة Live Cockpit مصممة كوحدات مستقلة وقابلة للإخفاء، وهو أساس جيد للتفاعل المستقبلي.

## ما هو ضعيف

1. `LiveCockpit` يعتمد حالياً على بيانات `DEMO / LOCAL ONLY`؛ الخريطة ليست خريطة جغرافية حقيقية.
2. لا توجد حالياً طبقة مزود جغرافي قابلة للتهيئة لـGoogle Maps/Mapbox أو GPS مع سياسة خصوصية وموافقة.
3. بعض KPIs تفاعلية بصرياً لكنها ليست كلها مربوطة بتحديث لحظي أو مصدر بيانات فعلي.
4. Profiles التمويل والقطاعات مرجعية وليست متطلبات رسمية موثقة لجهة محددة.
5. التصدير PDF يعتمد في الإنتاج على محرك خلفي يجب تثبيته وتشغيله كخدمة، وليس على جهاز العميل.
6. لا توجد بعد دورة كاملة موثقة للتشغيل متعدد المؤسسات من التسجيل إلى العزل والتدقيق في بيئة إنتاج.

## ما هو حرج

### P0 — يمنع الإطلاق الخارجي

- تثبيت نموذج تفويض متعدد المؤسسات واختباره ضد تسرب البيانات.
- إكمال بوابة الأمن والاستعادة والسجلات قبل تفعيل أي وصول خارجي.
- منع أي رقم أو خريطة أو مؤشر تجريبي من الظهور كحقيقة إنتاجية.
- تثبيت محرك PDF خلفي ومراقبته، مع اختبار الخطوط العربية والتخطيط.

### P1 — يمنع تسويق المنصة كمنتج مكتمل

- تحويل Live Cockpit من بيانات تجريبية إلى مصادر محلية موثقة أو إبقاؤه معلماً بوضوح كاستكشاف فقط.
- إضافة حالات تحميل/فشل/انقطاع API لكل وحدة تفاعلية.
- ربط التقرير التمويلي بمدخلات نموذج جمع بيانات المشروع الفعلي.
- تعريف Profile رسمي فقط عند وجود مرجع جهة مراجع، وعدم استخدام أسماء الجهات تسويقياً بلا سند.

### P2 — تحسين تنافسي

- Drag-and-drop محفوظ لكل مستخدم.
- Drill-down أعمق لمؤشرات المالية والمخاطر.
- قوالب قطاعات أكثر، ومقارنة نسخ مرئية للتقرير.
- مراقبة أداء الواجهة وإمكانية الوصول واختبارات المتصفحات.

## الأولوية الموصى بها

1. **حزمة الأمان والتشغيل والإطلاق المحلي**: إغلاق P0.
2. **ربط نموذج جمع البيانات بالتقرير**: تحويل العينة إلى مسار مستخدم حقيقي.
3. **فصل Demo عن Production بالسياسة والواجهة والعقد**.
4. **تثبيت طبقة الخرائط كمزود اختياري** بعد وجود الموافقة والخصوصية والمصدر.
5. **اختبار قبول خارجي محدود** على مشاريع حقيقية ببيانات محلية قبل أي وعود سوقية.

## القرار

ASIE قوية كمنصة تحليل وقرار محلية ذات حدود حقيقة واضحة، وليست ضعيفة في جوهرها. الخطر الأكبر ليس الحسابات الحالية؛ بل أن يفسر المستخدم عناصر العرض التجريبية أو Profiles المرجعية على أنها بيانات سوق حقيقية أو قبول تمويلي. لذلك الأولوية ليست إضافة مزايا أكثر، بل إغلاق فصل Demo/Production، الأمن التشغيلي، وربط بيانات العميل الحقيقية.
