# ASIE PR-03 — سجل مركز القيادة ومساحة القرار

## الحالة

- **الحالة:** منفذ ومتحقق محلياً.
- **النطاق:** `PR-C3 + PR-C4` فقط.
- **مصدر العرض:** إسقاطات API/Snapshot/Report/Decision Pack؛ لا حسابات أو verdict أو readiness في React.
- **الوصول الخارجي:** معطل.

## ما نُفذ

### Client Command Center

- أولويته هي: حالة القرار الحالية، سببها، والإجراء التالي الواحد.
- الإجراء التالي مشتق من حالة الخادم: تعريف مشروع، حفظ/فحص الجاهزية، معالجة العوائق، تشغيل التحليل، أو مراجعة حزمة القرار.
- يعرض مؤشرات القرار المتاحة فقط من Snapshot: NPV وIRR وفترة الاسترداد واحتياج التمويل واحتمال الجدوى حين تكون موجودة.
- القيم غير المتاحة لا تصبح صفراً أو تقديراً بصرياً؛ تظهر مع سبب عدم التوفر.
- يعرض جاهزية القرار وثقة الأدلة والمخاطر وسجل اللقطات، مع روابط منضبطة إلى الأدلة والتقرير وحزمة القرار.

### Decision Workspace

- يعرض Sovereign Verdict وسببه ومرجع Snapshot/Run، ثم مذكرة Decision Pack المحفوظة.
- يفصل الأدلة والمخاطر وخطة التنفيذ والمراجعة البشرية والإجراءات في أقسام مستقلة.
- Human Review يبقى overlay منفصلاً؛ نص الواجهة يوضح أنه لا يعدل Snapshot أو hash أو الحكم.
- لا تعرض مساحة العميل المعمارية الداخلية كأداة قرار؛ تظل شاشة runtime الحالية تفصيلاً محلياً منفصلاً حتى PR-B1.

## التحقق

- معاينة محلية للحالة قبل Snapshot ولحالة مشروع يملك Snapshot وDecision Pack محفوظين.
- `pnpm build`، suite الاختبارات، compile backend، وفحص Freeze تجرى كدليل إغلاق الحزمة.
