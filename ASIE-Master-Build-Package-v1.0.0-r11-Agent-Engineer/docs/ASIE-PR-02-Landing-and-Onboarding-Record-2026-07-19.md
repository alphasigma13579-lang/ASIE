# ASIE PR-02 — سجل صفحة الهبوط والإدخال

## الحالة

- **الحالة:** منفذ ومتحقق محلياً.
- **النطاق:** `PR-C1 + PR-C2` فقط.
- **لم يتغير:** AAS Runtime Freeze v1.0 أو API contracts أو منطق الحساب/الجاهزية/القرار أو Snapshot.
- **الوصول الخارجي:** معطل؛ لا دفع أو API حكومي أو مزود AI أو شبكة خارجية.

## ما نُفذ

### صفحة هبوط عربية أولاً

- وعد منتج يركز على قابلية مراجعة القرار، لا على أرقام أو عملاء أو ادعاءات دقة.
- تصور حركي أصلي لطبقات الدليل والجاهزية وSnapshot؛ وهو موسوم صراحة بأنه توضيحي ولا يمثل مشروعاً حقيقياً أو بيانات قرار.
- شريط خدمات يشرح الإدخال الموجّه، تتبع الأدلة، الجاهزية الصريحة، والمراجعة البشرية.
- رحلة قرار مفهومة: تعريف المشروع → ربط الأدلة → التحقق من الجاهزية → التشغيل والمراجعة.
- إشارات ثقة مرئية: لا شبكة خارجية، أدلة محلية، وSnapshot غير قابل للتغيير.
- مراعاة `prefers-reduced-motion` لإيقاف الحركة عند طلب المستخدم ذلك.

### الإدخال

- CTA لا يفتح اشتراكاً أو تجميع بيانات؛ ينقل المستخدم إلى بوابة محلية واضحة تشرح الخصوصية والملكية وحدود القرار.
- بعد الموافقة ينتقل المستخدم مباشرة إلى «معالج المشروع» بدلاً من لوحة القيادة، محافظةً على رحلة:
  `إعداد المشروع → الموقع → القطاع → التصنيف → المشكلة → الجمهور → المالية → الأدلة → التشغيل`.
- واجهة React لا تحسب مالياً ولا تنشئ حكماً أو readiness؛ الحفظ والتشغيل يظلان عبر API والخلفية.
