# ASIE Algorithm Catalog v1.0.0 Full

## كتالوج خوارزميات ASIE v1.0.0 النسخة الكاملة

This catalog defines official algorithms used by ASIE v1.0.0.

هذا الكتالوج يحدد الخوارزميات الرسمية المستخدمة في ASIE v1.0.0.

## Rule

## القاعدة

If an algorithm is not listed here, KIMI must stop and ask.

إذا لم تكن الخوارزمية مذكورة هنا، يجب على KIMI أن يتوقف ويسأل.

Chart datasets are algorithms too. KIMI must not treat charts as decorative UI.

بيانات الشارتات خوارزميات أيضًا. ممنوع على KIMI التعامل مع الشارتات كزينة واجهة.

Project dashboard scores, confidence indicators, risk matrices, scenario comparisons, strategic frameworks, and investor-readiness views must use `DASH-ALG-01` through `DASH-ALG-12`. The dashboard composes owner outputs and never becomes a new truth owner.

درجات لوحة المشروع ومؤشرات الثقة ومصفوفات المخاطر ومقارنات السيناريوهات والأطر الاستراتيجية والجاهزية الاستثمارية تستخدم `DASH-ALG-01` إلى `DASH-ALG-12`. تجمع اللوحة مخرجات المالكين ولا تصبح مصدر حقيقة جديدًا.

Professional feasibility uses `FST-ALG-01` through `FST-ALG-03`, `FIN-ALG-06` through `FIN-ALG-12`, `PROC-ALG-01` through `PROC-ALG-02`, and `METH-ALG-01` through `METH-ALG-02`. These extend existing owner Modules and do not create a new architecture component.

دراسة الجدوى الاحترافية تستخدم خوارزميات العمق والاكتمال والمطابقة، والتحليل المالي المتقدم، وضبط المشتريات والمراجع المنهجية. وهي امتداد للمالكين الحاليين وليست مكونًا معماريًا جديدًا.

MOF/Etimad references are official for applicable procurement workflows, not proof of one universal official feasibility template. International and commercial references are methodology references only.

Google Analytics and Zoho Analytics are product analytics sources only. They do not produce market evidence or finance inputs.

Google Analytics وZoho Analytics مصادر تحليلات منتج فقط. لا تنتج أدلة سوق ولا مدخلات مالية.

Public evidence must be fetched by ASIE. The user must not be asked to search for public reports.

الأدلة العامة يجب أن يجلبها ASIE. لا يطلب من المستخدم البحث عن التقارير العامة.

Human verification, Authenticator MFA, notification preferences, Telegram/WhatsApp opt-in, and advanced admin operations are controlled algorithms, not UI-only features.

التحقق من الإنسان، MFA عبر Authenticator، تفضيلات التنبيهات، موافقة تليجرام/واتساب، وعمليات الإدارة المتقدمة خوارزميات مضبوطة وليست ميزات واجهة فقط.

Sector taxonomy is mandatory. Every project must map to a primary sector, subsector, source plan, investment indicators, and sector-specific evaluation criteria.

تصنيف القطاعات إلزامي. كل مشروع يجب أن يرتبط بقطاع رئيسي، تصنيف فرعي، خطة مصادر، مؤشرات استثمار، ومعايير تقييم خاصة بالقطاع.

## Algorithm Record Format

## صيغة سجل الخوارزمية

Each algorithm must define:

كل خوارزمية يجب أن تحدد:

- ID / المعرف.
- Name / الاسم.
- Arabic / الترجمة.
- Owner Component / الموديول المالك.
- Purpose / الهدف.
- Inputs / المدخلات.
- Outputs / المخرجات.
- Deterministic Status / الحالة الحتمية.
- AI Role / دور الذكاء الاصطناعي.
- Steps / الخطوات.
- Failure Conditions / حالات الفشل.
- Audit Events / أحداث التدقيق.
- Acceptance Tests / اختبارات القبول.
- Forbidden Behavior / السلوك الممنوع.
