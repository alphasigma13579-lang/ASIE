# Smart Site and Dynamic Charts

## الموقع الذكي والشارتات الديناميكية

## Core Principle

## المبدأ الأساسي

ASIE is an intelligent platform, not a normal website with AI decoration in the interface.

ASIE منصة ذكية، وليست موقعًا عاديًا يضع الذكاء الاصطناعي كزينة في الواجهة.

Intelligence must exist across:

يجب أن يكون الذكاء موجودًا عبر:

- Guided project understanding / فهم المشروع الموجه.
- Market evidence generation / إنتاج أدلة السوق.
- Deterministic finance / التمويل الحتمي.
- Decision deliberation / مداولات القرار.
- Dynamic reports / التقارير الديناميكية.
- Interactive visual analytics / التحليلات المرئية التفاعلية.
- Admin intelligence / ذكاء الإدارة.
- Audit and observability intelligence / ذكاء التدقيق والمراقبة.

## Smart Site Definition

## تعريف الموقع الذكي

A screen is smart only if it uses verified platform outputs, reacts to user choices, exposes evidence, and updates charts or recommendations through controlled contracts.

تكون الشاشة ذكية فقط إذا استخدمت مخرجات موثقة من المنصة، وتفاعلت مع اختيارات المستخدم، وعرضت الأدلة، وحدثت الشارتات أو التوصيات عبر عقود مضبوطة.

AI is one capability inside the system, not the whole intelligence.

الذكاء الاصطناعي قدرة واحدة داخل النظام، وليس كل الذكاء.

## Dynamic Chart Rules

## قواعد الشارتات الديناميكية

- Charts must be generated from approved data outputs only.
- الشارتات تولد من مخرجات بيانات معتمدة فقط.

- Chart numbers must come from `Market Intelligence Module`, `Finance Engine Module`, `Decision Council Module`, `Subscription / Usage Module`, or `Audit / Observability Module`.
- أرقام الشارتات تأتي من موديول ذكاء السوق، محرك التمويل، مجلس القرار، الاشتراكات، أو التدقيق والمراقبة.

- AI may summarize charts but must not generate chart data.
- الذكاء الاصطناعي يجوز أن يلخص الشارت، لكنه لا يولد بيانات الشارت.

- Every chart must show source or calculation basis where relevant.
- كل شارت يعرض المصدر أو أساس الحساب عند الحاجة.

- Empty, loading, error, insufficient data, and blocked states are required.
- حالات الفراغ، التحميل، الخطأ، نقص البيانات، والمنع مطلوبة.

## Page-Level Chart Requirements

## متطلبات الشارتات حسب الصفحة

### Landing Page

### صفحة الهبوط

Allowed charts:

الشارتات المسموحة:

- Simplified product examples / أمثلة مبسطة للمنتج.
- No fake business claims / لا ادعاءات تجارية وهمية.
- Curated screenshots from internal ASIE charts / لقطات منتقاة من شارتات ASIE الداخلية.
- Sanitized sample datasets / بيانات عينة منقحة.

Landing Page Preview Rules:

قواعد لقطات صفحة الهبوط:

- The landing page may use selected screenshots from real ASIE report models, chart models, dashboard models, and analysis outputs.
- يجوز لصفحة الهبوط استخدام لقطات مختارة من نماذج تقارير ASIE، نماذج الشارتات، نماذج اللوحات، ومخرجات التحليل.

- Screenshots must never expose real user data, project names, phone numbers, emails, coordinates, payment data, supplier documents, or private organization details.
- يمنع أن تكشف اللقطات بيانات مستخدم حقيقية، أسماء مشاريع، أرقام تواصل، بريد، إحداثيات، بيانات دفع، مستندات موردين، أو تفاصيل منظمة خاصة.

- Landing screenshots must be tagged as one of:
- يجب وسم لقطات صفحة الهبوط بإحدى الحالات:

  - `demo_sample` / عينة تجريبية.
  - `anonymized_real_output` / مخرج حقيقي مخفي الهوية.
  - `generated_product_mock` / نموذج منتج مولد.

- If the screenshot uses sample data, it must not imply guaranteed revenue, guaranteed success, or official approval.
- إذا استخدمت اللقطة بيانات عينة، فيمنع أن توحي بضمان الإيراد أو النجاح أو الموافقة الرسمية.

- Landing preview charts should show product capability, not final investment promises.
- شارتات صفحة الهبوط تعرض قدرة المنتج، لا وعودًا استثمارية نهائية.

Recommended Landing Preview Assets:

الأصول المقترحة لصفحة الهبوط:

- Feasibility score gauge / عداد درجة الجدوى.
- Competitor map preview / معاينة خريطة المنافسين.
- Cost breakdown donut / شارت حلقي لتقسيم التكلفة.
- Cash flow line chart / خط التدفق النقدي.
- Evidence pack source bars / أعمدة مصادر الأدلة.
- Decision council vote preview / معاينة أصوات مجلس القرار.
- Risk heatmap / خريطة حرارة المخاطر.
- Report page screenshot / لقطة صفحة التقرير.

## Interaction Requirements

## متطلبات التفاعل

Every analytical chart should support where relevant:

كل شارت تحليلي يدعم عند الحاجة:

- Filter by date / فلترة بالتاريخ.
- Filter by project / فلترة بالمشروع.
- Filter by region / فلترة بالمنطقة.
- Filter by sector / فلترة بالقطاع.
- Drill-down / النزول للتفاصيل.
- Tooltip with source / Tooltip بالمصدر.
- Export image or report section / تصدير صورة أو قسم تقرير.
- RTL labels / تسميات RTL.
- Arabic number formatting / تنسيق أرقام عربي.

## Forbidden Chart Behavior

## سلوك الشارتات الممنوع

- Fake demo numbers in production.
- أرقام تجريبية وهمية في الإنتاج.

- AI-generated chart data.
- بيانات شارت مولدة من AI.

- Chart without owner module.
- شارت بلا موديول مالك.

- Chart without empty/error state.
- شارت بلا حالة فراغ أو خطأ.

- Chart implying unsupported precision.
- شارت يوحي بدقة غير مدعومة.

## Project Dashboard Binding Reference

## المرجع الملزم للوحة المشروع

All project-facing charts, scores, metric cards, strategic frameworks, risk matrices, scenario comparisons, decision-persona views, investment outputs, and exports must follow `Project-Intelligence-Dashboard-and-Visual-Outputs.md` and `DASH-ALG-01` through `DASH-ALG-12`.

The dashboard composes approved owner outputs. It must not recalculate, infer missing numbers, convert confidence into success probability, or copy data and claims from legacy screenshots.
