🏗️ Backend Stack الكامل لمشروع ASIE
بناءً على وثائق المشروع الـ 17 التي قرأتها، إليك القرارات التقنية الدقيقة:
1. 🗄️ Database
Table
المكون	التقنية	السبب
الأساسي	PostgreSQL 16	Relational + JSON support + Full-text search
Realtime	Supabase	WebSocket subscriptions + Presence API
Vector DB	pgvector (inside PostgreSQL)	تخزين embeddings للبحث الذكي
Cache	Redis	Sessions + Rate limiting + Hot data
File Storage	Supabase Storage	Reports PDF/PPT + User uploads
Backup	pg_dump + S3	Daily automated backups
plain
📁 PostgreSQL
 ├── users, projects, reports (tables)
 ├── pgvector (AI embeddings)
 └── Row Level Security (RLS)
 
📁 Redis
 ├── Sessions (JWT blacklist)
 ├── Rate limiting counters
 └── Chat message queue
 
📁 Supabase
 ├── Realtime subscriptions
 ├── Auth integration
 └── File storage
2. 🔐 Auth (المصادقة)
Table
المكون	التقنية	الوضع
الأساسي	Supabase Auth	OAuth 2.0 + JWT + Refresh tokens
Email/Password	Supabase + bcrypt	مع تفعيل البريد
Social Login	Google + Apple + LinkedIn	OAuth 2.0
RBAC	مخصص (Casbin)	5 أدوار: User → Admin
2FA	TOTP (Google Authenticator)	Planned for Pro tier
Session	Redis + JWT	24h access / 7d refresh

3. 📨 Messaging (الرسائل والأحداث)
Table
المكون	التقنية	الاستخدام
Event Bus	Internal (Python asyncio)	أحداث داخل التطبيق
Queue	Redis Pub/Sub	Chat messages + AI requests
Background Jobs	Celery + Redis	PDF generation + Email sending
Real-time	Supabase Realtime	Push notifications + Chat
WebSocket	FastAPI WebSocket	Chat streaming (AI responses)
plain
📨 تدفق الرسائل:
User Chat → FastAPI WebSocket → Redis Queue → Celery Worker → AI API
                                     ↓
                              Supabase Realtime (push to UI)
4. 🌐 Hosting (الاستضافة)
Table
المكون	التقنية	الموقع
Frontend	Vercel	Edge CDN (أقرب للمستخدم)
Backend API	VPS (Saudi Telecom Cloud)	داخل السعودية (SDAIA compliance)
AI Models	Saudi VPS + GPU	Kimi/Llama محليًا
Database	Supabase Cloud	Managed PostgreSQL
Files	Supabase Storage	CDN للملفات
Docker	Docker Compose	Containerization للخدمات
plain
🌍 Architecture موزعة:

[User] → [Vercel CDN] → [Saudi VPS API]
                              ↓
                    [Supabase DB] ← [Redis Cache]
                              ↓
                    [AI Cluster] (GPU VPS)
لماذا VPS Saudi Telecom وليس AWS/GCP؟
✅ SDAIA compliance (البيانات لا تغادر السعودية)
✅ Cloud Act (تجنب وصول الحكومة الأمريكية)
✅ GDPR (حماية بيانات EU users)
✅ Latency أقل للمستخدمين السعوديين
