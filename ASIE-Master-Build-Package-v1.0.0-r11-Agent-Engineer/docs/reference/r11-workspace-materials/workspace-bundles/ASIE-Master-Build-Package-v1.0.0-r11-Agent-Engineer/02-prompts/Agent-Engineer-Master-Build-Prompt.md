Do not begin implementation until the preflight checklist passes.

لا تبدأ التنفيذ حتى ينجح فحص ما قبل البناء.

Do not claim legal certification. Do not use AI as legal authority. Do not implement a generic ministry crawler or any crawler under the strict profile. Every enabled source must be an exact official open dataset or open API with current reusable-data evidence.

Do not provide source-page text to AI. Strategy alignment uses metadata and approved ASIE-authored cards only. A source title and official URL are citations, not permission to copy its prose, images, taxonomy, or layout.

Do not create a Dashboard Module or Dashboard Layer. The frontend must use React + TypeScript + Vite + pnpm and consume TypeScript-safe view contracts. It must not calculate finance, generate chart values, call AI or data sources directly, or infer missing results.

Do not create a Feasibility, Procurement, or Methodology Module. Do not claim that MOF/Etimad provides one universal official feasibility template. Do not use a generic procurement form when an exact competition package exists. Do not provide Aljdwa page content to AI or automated retrieval. AI may consume only approved original ASIE methodology cards.

Read `06-integration/ASIE-r10-Correction-Closure-and-Traceability.md`, `06-integration/ASIE-Feasibility-Decision-Integration-Addendum-v1.0.0.md`, and `07-architect-decisions/ASIE-r11-Architect-Resolution-Closure.md` before implementation. They are binding for Saudi feasibility decision behaviour and architect-resolution behaviour. Labels such as `ASIE.MOD.RISK` are logical capabilities mapped to existing approved Modules, never authorization to create a physical Module. Monte Carlo is mandatory in Finance Engine and the first project page shows the probability of passing feasibility gates or `NOT_READY` with reason; never call it a success probability.

The Five Sovereign Decision Protocol executes in fixed isolation: Project Manager -> Business Advisor -> Technical Auditor -> Analyst Coach -> Resistance Test -> Sovereign Verdict. No persona sees another persona output. Persona KPIs and the Sovereign Verdict are deterministic. `DEC-ALG-03` consensus/vote diagnostics are advisory only and never determine, override, or appear as the Sovereign Verdict.

Before any build phase is accepted, execute every `R10-AC-*` rejection test named in the r10 correction-closure matrix. A passing visual screen, mock number, plan entitlement, or AI explanation is not evidence of compliance. Record the immutable calculation snapshot ID, algorithm version, evidence snapshot ID, and test result for every accepted feasibility decision.

Before any architect optimization or UX acceleration is accepted, execute every `R11-AC-*` rejection test named in the r11 architect-resolution closure. Shared memory, bulk transfer, sensitivity previews, remediation envelopes, synthetic baselines, named AI providers, and benchmark claims are implementation mechanisms only. They must not change AAS component ownership, message path, trust model, calculation ownership, FSDP isolation, source licensing, or immutable snapshot parity.
