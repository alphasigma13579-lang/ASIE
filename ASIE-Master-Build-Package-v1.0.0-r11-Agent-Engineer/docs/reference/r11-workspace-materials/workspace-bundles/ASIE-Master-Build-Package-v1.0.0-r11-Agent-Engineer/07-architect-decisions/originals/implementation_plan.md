## مراجعة المستخدم مطلوبة (User Review Required)

> \\\\\\\[!IMPORTANT]
> - \\\\\\\*\\\\\\\*ملف البيانات المفتوحة والصناعية المعتمدة:\\\\\\\*\\\\\\\* سيتم تجميع بيانات السوق الحقيقية دورياً عبر عمليات خلفية مجدولة (Batch Jobs) باستخدام إيجنت KIMI ومفتاح Tevali لتشكيل فهارس مرجعية معتمدة بشرياً. لا يتم أي زحف مباشر في وقت تشغيل المستخدمين لحماية الملكية الفكرية والالتزام بالقوانين.
> - \\\\\\\*\\\\\\\*الذاكرة المشتركة لحل بطء ناقل النظام:\\\\\\\*\\\\\\\* سنستخدم تقنية الذاكرة المشتركة (Zero-Copy Shared Memory) لتبادل الرسائل الكبيرة بين الوحدات برمجياً تحت مظلة طبقة عقود السوكيت (SCL).
> - \\\\\\\*\\\\\\\*حلقات المواءمة ومصفوفة الحساسية:\\\\\\\*\\\\\\\* يتم إرسال مصفوفات مسبقة الحساب (Sensitivity Matrix) لواجهة React لتسريع استجابة السلايدرات بصررياً، وفي حال الرفض التلقائي من شخصيات FSDP، يتم إرجاع "مظروف تصحيحي" يوجه المستخدم بوضوح لكيفية تعديل أرقامه.

### المكون 3: محرك التمويل الحتمي ومصفوفة الحساسية

دمج الخوارزميتين `FIN-ALG-04` و `FIN-ALG-12` مع حساب مصفوفة الحساسية المسبقة للواجهة الأمامية.

#### \[NEW] \[finance\_engine.py](file:///d:/منصة ASIE مشروع/src/backend/modules/finance\_engine.py)

## المرحلة الثانية: نظام الشخصيات التكيفي ومحرك الاشتراكات (Phase 2 — Post Core Delivery)

> \\\\\\\[!NOTE]
> هذه المرحلة \\\\\\\*\\\\\\\*مؤجلة عمداً\\\\\\\*\\\\\\\* حتى اكتمال المرحلة الأولى واجتياز اختبارات r10. لا يتم البدء بها إلا بعد التحقق من استقرار نواة المنصة.

### القرار الاستراتيجي المسجّل (Architectural Decision Record)

**القرار:** نظام الشخصيات التكيفي هو **طبقة عرض (Display Layer) فوقية** ولا يؤثر على معمارية النواة.
**السبب:** المنصة تحسب نفس الأرقام لجميع المستخدمين. الفرق الوحيد هو كيف تُعرض هذه الأرقام وما هي الروابط والتوجيهات المصاحبة لها.
**الفائدة التجارية:** نظام الشخصيات هو محرك الإيرادات (Revenue Engine) لنموذج الاشتراكات.

### مكونات المرحلة الثانية

#### \[NEW] نظام تحديد الشخصية (Persona Detection Engine)

تحديد مستوى المستخدم عبر 3 أسئلة فقط عند التسجيل، وتصنيفه في إحدى الشخصيات الأربع:

|الشخصية|نموذج الاشتراك|المميزات|
|-|-|-|
|🟢 المستكشف|مجاني محدود|تقرير واحد، KPIs أساسية، شرح مبسط|
|🔵 المبادر|اشتراك شهري منخفض|نصائح موجهة + روابط الجهات الحكومية|
|🟡 الخبير الفردي|اشتراك شهري متوسط|تحليل عميق + حساسية + مقارنة بالسوق|
|🔴 المكتب الاستشاري|اشتراك مؤسسي سنوي|API Access + مطابقة كراسات الشروط|

#### \[NEW] بطاقات التوصيات الفاخرة الموجهة (Actionable Recommendation Cards)

كل توصية تحتوي على:

* شارة المشكلة مع وصف مبسط أو تقني بحسب مستوى المستخدم.
* أزرار إجرائية تنقل المستخدم فوراً لمساحة العمل الصحيحة داخل المنصة.
* روابط موثقة للجهات الحكومية الخارجية المختصة (Etimad، Monsha'at، GASTAT، وزارة الصناعة).
* شروحات بصرية تفاعلية قصيرة للمبتدئين فقط.

#### \[NEW] نظام الاشتراكات والدفع (Subscription \& Billing Module)

* مبني على نموذج Freemium محلي متوافق مع قوانين المعاملات الإلكترونية السعودية.
* الاشتراك لا يغير حسابات المنصة أو يخفي تحذيرات حرجة (ضابط r10 إلزامي).
* يتحكم فقط في عمق التقارير وعدد المشاريع وصلاحيات التصدير.
