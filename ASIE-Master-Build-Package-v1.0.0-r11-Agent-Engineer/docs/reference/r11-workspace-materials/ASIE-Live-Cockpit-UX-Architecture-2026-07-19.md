# ASIE — معمارية تجربة Live Cockpit

## قرار التصميم

تتحول مساحة العميل من صفحات تقنية إلى سطح عمل حيّ حول سؤال واحد:

> ماذا أفعل بمشروعي الآن، وما الدليل الذي يدعم الخطوة التالية؟

كل Widget مستقلة في العرض والتحكم، لكنها لا تملك حقيقة مستقلة: تقرأ من API/Snapshot أو من سجل بيانات محلي موثق.

## الهيكل البصري

```
Smart Input Hub
  القطاع + نوع النشاط + الدولة/المنطقة + موقع مختار + حالة البيانات

Live Workspace
  ┌ قرار اليوم ────────────┐  ┌ الخريطة السوقية ───────────┐
  │ الحكم / السبب / CTA    │  │ منافسون ومواقع من بيانات   │
  │                         │  │ محلية مرخصة أو حالة فارغة │
  └─────────────────────────┘  └────────────────────────────┘
  ┌ KPIs قابلة للتفصيل ────┐  ┌ خطة التنفيذ والدعم ─────────┐
  │ كل KPI → مصدره/تفصيله │  │ تمويل/ترخيص/خطوة/مالك      │
  └─────────────────────────┘  └────────────────────────────┘

Reports Drawer: تقرير القرار، Snapshot، خطة التنفيذ، سجل التغيير
```

## Widgets المستقلة

| الوحدة | التفاعل | مصدر الحقيقة | حالة عدم البيانات |
|---|---|---|---|
| قرار اليوم | فتح السبب والتقرير وتنفيذ CTA | Snapshot/Decision Pack | «أكمل المدخلات لتكوين قرار» |
| KPI Widget | drill-down إلى formula/evidence/risk | output محفوظ في Snapshot | لا تعرض قيمة تقديرية |
| خريطة السوق | filter حسب القطاع/المنطقة، zoom، select marker | `Local Market Evidence Pack` فقط | «أضف/استورد بيانات مواقع مرخصة» |
| موقع مرشح | مقارنة مواقع يدخلها العميل | Site Candidate Evaluation + evidence | «لا توجد بيانات كافية لتوصية موقع» |
| منافسون | تصفية/فرز/فتح مصدر كل منافس | Local competitor records | «شدة المنافسة تحتاج دليلاً» |
| دعم وإجراءات | checklist حسب النشاط والمنطقة | Guidance Catalog مراجع محلياً | «المسار يحتاج مراجعة مشغل» |
| مشاريع مشابهة | حالات منشورة بموافقة فقط | Consent-based catalog | «لا توجد حالات منشورة» |
| تقارير | تصدير projection محفوظ | Snapshot/Report/Decision Pack | لا يوجد قبل التشغيل |

## Smart Input Hub

شريط ثابت أعلى مساحة العميل، لا يجبر العميل على رؤية المعالج كله:

1. **ماذا تريد أن تبني؟** قطاع ونشاط دقيق.
2. **أين؟** دولة، منطقة، مدينة، ثم موقع يدخله العميل يدوياً في النسخة المحلية.
3. **ما غرضك الآن؟** قرار جدوى / موقع / منافسين / دعم وإجراءات.
4. **حالة البيانات:** محفوظ، يحتاج دليل، جاهز للتشغيل.

لا نطلب GPS ولا نرسل موقع المستخدم. GPS وGoogle Maps Adapter يظلان `DISABLED` إلى أن تمر P0 وتصدر موافقة صريحة.

## استقلالية العرض

- المستخدم يستطيع إخفاء/إظهار Widgets وإعادة ترتيبها؛ التفضيل محلي في المتصفح ولا يغير Snapshot.
- لكل Widget عنوان، مصدر، وقت آخر تحديث، وحالة بيانات نصية لا تعتمد على اللون.
- الضغط على KPI أو marker لا يحسب شيئاً في React؛ يفتح drill-down من API.
- Drag-and-drop يجب أن يبقى قابلاً للوصول عبر لوحة المفاتيح (Move earlier/later)، لا بالسحب فقط.

## الحدود الأمنية والواقعية

- Google Maps API وGPS وبيانات منافسين حية: **غير منفذة وغير مسموحة الآن**.
- لا تُشارك مشاريع منظمات أخرى في «مشاريع مشابهة» إلا بموافقة نشر صريحة وبعد إزالة الهوية.
- مسارات التأشيرات والتراخيص والدعم معلومات مرجعية مراجعَة، وليست قراراً حكومياً أو قانونياً.
- أفضل موقع/شارع ليس حكماً من الواجهة: يحتاج سجلاً محلياً بأدلة الموقع ومعايير تقييم معلنة.

## وضع المحاكاة التطويري المعتمد

في بيئة التطوير المحلية يجوز تمثيل بيانات يُفترض أنها آتية من مصادر خارجية—مثل المنافسين، الكثافة، الإيجارات، المسارات الحكومية، وإشارات الطلب—من دون أي اتصال فعلي.

كل سجل محاكى يحمل إلزامياً:

```text
data_mode: demo_simulated_external
display_badge: DEMO / LOCAL ONLY
assumed_source: اسم المصدر المفترض أو فئته
observed_at: تاريخ افتراضي أو تاريخ seed
confidence: تجريبي
production_admission: blocked
```

هذه البيانات مطلوبة لتطوير التفاعل والتصفية والخرائط والتقارير والاختبارات. لكنها لا تصبح حقيقة قرار ولا تدخل Snapshot/Report إنتاجياً إلا بعد استبدالها بسجل مصدر حقيقي مراجع. بهذا نختبر المنتج كاملاً من دون فتح شبكة أو كسر حدود المعمارية أو إنتاج حسابات غير معلومة المصدر.
