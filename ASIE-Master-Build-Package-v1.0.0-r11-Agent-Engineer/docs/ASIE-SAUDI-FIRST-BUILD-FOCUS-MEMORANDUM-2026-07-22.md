## 1. القرار التنفيذي الأعلى

الأولوية الحالية هي **إنهاء بناء منصة ASIE وتشغيل رحلة مستخدم سعودية كاملة قابلة للاختبار**. لا يكون توسيع الوثائق أو إعادة صياغة الرؤية أو إضافة قدرات مستقبلية أولوية ما لم يكن ذلك شرطًا مباشرًا للتنفيذ أو الاختبار أو القبول.

معيار التقدم ليس عدد الملفات أو الوحدات أو الشاشات، بل قدرة مستخدم داخل المملكة على:

1. تعريف مشروع حقيقي وإدخال بياناته وأدلته.
2. معرفة البيانات الناقصة والافتراضات المستخدمة.
3. الحصول على تحليل مالي وسوقي ومخاطر محكوم.
4. فهم مصدر كل رقم وحدود كل استنتاج.
5. الوصول إلى قرار واضح وخطة تنفيذ وSnapshot ثابتة قابلة للتتبع.

---

## 2. النطاق الجغرافي والسوقي الملزم للمرحلة الأولى

المملكة العربية السعودية هي **السوق والنطاق التشغيلي الوحيد للمرحلة الأولى**.

- المنتج الحالي مصمم للمستخدم والسوق والأنظمة والبيانات السعودية.
- التوسع الخليجي أو العربي أو العالمي رؤية مستقبلية مشروطة بإثبات النجاح داخل المملكة، وليس جزءًا من نطاق البناء الحالي.
- لا يجوز توسيع المتطلبات الحالية لتغطية أسواق أخرى على حساب اكتمال المسار السعودي.
- اختلاف المناطق والمدن والقطاعات السعودية يدخل في السياق المحلي عندما تتوفر له بيانات موثوقة.

### المستخدمون المستهدفون

- المبادر وصاحب فكرة المشروع.
- رائد الأعمال والمنشأة الناشئة.
- المنشآت الصغيرة والمتوسطة.
- المستثمر والجهة التمويلية ضمن حدود المنتج والعقود المعتمدة.
- المستشار والحاضنة والمسرعة والمؤسسة عند تفعيل مساحات العمل المناسبة.

---

## 3. جوهر المنتج

ASIE ليست روبوت محادثة، وليست مولد تقارير، وليست ترجمة عربية لأداة أجنبية. هي **منصة سعودية لاستخبارات المشاريع والاستثمار ودعم القرار**، تحول مدخلات المستخدم والبيانات والأدلة إلى تحليل وقرار قابلين للتفسير والتتبع والتنفيذ.

القيمة المقصودة هي فهم المشروع **داخل السوق السعودي**، لا التعامل معه كنسخة مترجمة من مشروع في سوق أجنبي.

تميّز ASIE المستهدف عن أدوات مثل LivePlan وIdeaBuddy لا يعتمد على اللغة العربية وحدها، بل على:

- العمق الاقتصادي والتنظيمي والسوقي السعودي.
- الفصل بين الحقيقة والدليل والافتراض والحساب والتفسير والتوصية والحكم.
- تتبع المصادر وحداثتها ونطاقها الجغرافي.
- تحليل الفشل والمخاطر واختبارات الضغط، لا السيناريو المتفائل فقط.
- تحويل النتيجة إلى قرار وخطوة تنفيذية، لا الاكتفاء بوثيقة سردية.
- إخضاع الذكاء الاصطناعي للعقود والصلاحيات والتدقيق.

---

## 4. تطويع الذكاء الاصطناعي واللغة العربية

الهدف ليس مجرد إتاحة واجهة عربية، بل تطويع نماذج اللغة الكبيرة وأدوات الذكاء الاصطناعي لفهم المحتوى المهني والاقتصادي السعودي وصياغته بلغة عربية واضحة يفهمها المستخدم السعودي.

يشمل ذلك:

- المصطلحات التجارية والاقتصادية والتنظيمية المستخدمة في المملكة.
- سياق المبادر والمستثمر ورائد الأعمال والمنشآت الصغيرة والمتوسطة.
- الأنظمة والجهات والبرامج والحوافز والمصادر السعودية عند صلتها بالمشروع.
- التمييز بين المعلومة النظامية، والمؤشر الاقتصادي، والتوجه الاستراتيجي، والرأي التحليلي.
- إنتاج تفسير عربي واضح دون إخفاء عدم اليقين أو حدود الدليل.

### حدود سلطة الذكاء الاصطناعي

- الذكاء الاصطناعي يفسر ويصنف ويصوغ ويقترح ضمن الصلاحية الممنوحة.
- لا يملك الأرقام المالية، ولا ينشئ حقيقة سوقية، ولا يتخذ الحكم النهائي من تلقاء نفسه.
- الحسابات الحساسة تنفذها المحركات الحتمية المعتمدة.
- أي رقم أو ادعاء خارجي يحتاج مصدرًا أو وسمًا صريحًا بأنه افتراض.
- لا يجوز للـAI تجاوز Contracts أو Sockets أو APP أو Message Flow أو Zero Trust.
- لا يجوز إنشاء Runtime موازٍ للذكاء؛ `AAS governs execution. AIA governs intelligence.`

## 5. سياسة البيانات والمصادر السعودية

كون الجهة الناشرة دولية لا يجعل نطاق البيانات عالميًا داخل ASIE. يمكن استخدام البنك الدولي أو IMF أو OECD أو McKinsey أو غيرها عندما تكون البيانات أو الدراسة **خاصة بالسعودية أو ذات صلة سعودية موثقة**.

### ترتيب الاستخدام

1. المصادر الحكومية والتنظيمية والإحصائية السعودية الرسمية.
2. البرامج والمبادرات والجهات الوطنية السعودية ذات الصلة.
3. الأبحاث والبيانات السعودية الصادرة عن جهات موثوقة.
4. التقارير الدولية والاستشارية التي تتناول المملكة مباشرة.
5. المقارنات الدولية الموسومة، عند الحاجة فقط، دون إحلالها محل الدليل السعودي.

### متطلبات كل دليل

- المصدر والناشر.
- تاريخ النشر أو الاستخراج.
- حداثة البيانات وفترة القياس.
- النطاق الجغرافي والقطاعي.
- درجة الثقة وحدود الاستخدام.
- سلسلة الاشتقاق من المصدر إلى النتيجة.

لا تُسقط المتوسطات العالمية أو بيانات دولة أخرى على السوق السعودي تلقائيًا. وعند غياب البيانات السعودية، يجب إظهار الفجوة والافتراض ومنهج المقارنة بدل تقديمه كحقيقة سعودية.

## 6. رؤية السعودية 2030

رؤية 2030 طبقة سياق ومواءمة استراتيجية مهمة، لكنها ليست الهوية الوحيدة للمنصة ولا ضمانًا لنجاح المشروع.

يستخدم `Strategic Alignment Score` لشرح:

- الهدف أو البرنامج أو الاستراتيجية الوطنية ذات الصلة.
- طبيعة الارتباط: مباشر أو غير مباشر.
- الدليل والمؤشر المستخدمان.
- الفرصة أو الحافز المحتمل.
- المخاطر أو المتطلبات المصاحبة.

لا يجوز إعلان توافق مشروع مع رؤية 2030 بعبارة تسويقية عامة، ولا تحويل المواءمة الاستراتيجية إلى حكم تلقائي بالجدوى التجارية أو التمويلية.

## 7. الحوكمة المعمارية التي لا يجوز تجاوزها

- `AAS Runtime Freeze v1.0` هو خط التنفيذ المجمد.
- AIA معمارية تكاملية للذكاء ولا تكسر AAS ولا تنشئ Runtime ثانيًا.
- مسار التنفيذ يظل وفق الحدود والعقود المعتمدة في السجل القانوني للوثائق.
- لا توجد اتصالات مباشرة بين Modules؛ المرور عبر ASIE System Bus وSocket Contract Layer حسب المعمارية المعتمدة.
- Kernel لا يحتوي منطق أعمال أو مزودي AI أو تكاملات قابلة للاستبدال.
- قاعدة البيانات وتقنيات التنفيذ ومزودو النماذج ليست سلطة معمارية.
- أي تغيير يمس السطح المجمد يخضع لمسار ACR/IACR/ICCR المناسب، ولا يمر كتعديل تنفيذي صامت.

## 8. أولوية البناء الحالية

ترتب الأعمال بحسب المسار الحرج للوصول إلى منتج سعودي قابل للاستخدام:

1. تثبيت حالة `main` وتشغيل المشروع واختبار مسار Runtime الفعلي.
2. حصر المكتمل والناقص والمتعطل من الكود، لا من الوعود الوثائقية.
3. إكمال المحرك المالي الحتمي وصحة الحسابات الأساسية.
4. إكمال استقبال بيانات المشروع والملفات والأدلة والافتراضات.
5. إكمال التسعير والمصادر والبيانات السعودية المطلوبة للقرار.
6. ربط الواجهة العربية RTL بالخدمات الفعلية وإزالة المسارات الوهمية.
7. إكمال تحليل السوق والمخاطر والسيناريوهات واختبارات الضغط.
8. تشغيل الذكاء الاصطناعي المحكوم للتفسير والتوليف ضمن العقود المعتمدة.
9. إكمال مجلس القرار وSnapshot والتقارير وحزمة القرار.
10. اختبارات التكامل والأمن والأداء والقبول ثم تجربة سعودية فعلية.

لا يعني هذا الترتيب تفعيل عقد أو Module غير منفذ أو غير مجاز. المرجع في حالة التنفيذ هو Registry والكود واختبارات القبول، لا مجرد وجود وصف في AIA.

## 9. ما يؤجل الآن

ما لم يكن شرطًا للمسار السعودي الأول، يؤجل الآتي:

- التوسع الخليجي والعربي والعالمي.
- تعدد مزودي النماذج دون حاجة تشغيلية مثبتة.
- الزحف الواسع أو البنية المتجهية قبل اكتمال أساس القرار الحتمي ومصادره.
- الميزات التجميلية التي لا تزيل عائقًا من رحلة المستخدم.
- إنشاء وثائق أو Blueprints جديدة تكرر قرارات موجودة.
- أي قدرة مستقبلية لم تمر ببوابات العقود والاختبارات والتغيير المعتمدة.

---

## 10. معايير قبول المرحلة الأولى

تعد المرحلة الأولى جاهزة للتجربة عندما يثبت الاختبار أن:

- مستخدمًا سعوديًا يستطيع إكمال الرحلة الأساسية دون تدخل تقني يدوي.
- جميع الأرقام المهمة مصدرية أو محسوبة حتميًا أو موسومة كافتراضات.
- الأدلة السعودية مؤرخة وقابلة للتتبع.
- لا يولد النموذج اللغوي أرقامًا أو أحكامًا خارج صلاحياته.
- نتائج السوق والمال والمخاطر متصالحة بلا تناقضات صامتة.
- Snapshot لا تعاد كتابتها من الواجهة بعد اعتمادها.
- القرار النهائي مفهوم، وتظهر أسبابه وحدوده والخطوة التالية.
- السجلات الأمنية والتدقيقية تعمل وفق الحدود المعتمدة.
- لا توجد شاشات نجاح تعتمد على Mock Data أو مسارات غير متصلة في تجربة القبول.

## 11. افتراضات يجب إثباتها في السوق

هذه ليست حقائق محسومة، بل فرضيات تحتاج تجربة سعودية فعلية:

- استعداد المستخدم للدفع مقابل قرار محكوم بدل محادثة عامة مع نموذج لغوي.
- ثقة المستخدم والمؤسسة في منهجية المصادر وسلسلة الاشتقاق.
- تحقيق التوطين السعودي فرقًا ملموسًا في جودة القرار، لا فرقًا لغويًا فقط.
- قدرة رحلة الاستخدام على اختصار الوقت ورفع جودة القرار مقارنة بالبدائل الحالية.

تستخدم التجارب والمقابلات والاستخدام المدفوع أو الجاد للتحقق من هذه الفرضيات، ولا تستبدل بتقديرات نمو سوق الذكاء الاصطناعي العامة.

## 12. قاعدة التعامل مع هذه المذكرة

1. تقرأ عند بدء أي دورة بناء أو مراجعة منتج.
2. تستخدم لتحديد الأولويات ومنع الانحراف عن السعودية وإنهاء البناء.
3. لا تستخدم لإثبات أن مكونًا منفذ؛ الإثبات من الكود والاختبارات وAs-Built.
4. لا تعدل لتغيير المعمارية؛ يرفع التغيير عبر مسار الحوكمة المختص.
5. إذا تغير قرار منتجي مهم دون أثر معماري، يحدث سجل القرارات في إصدار جديد مع تاريخ وسبب واضحين.
6. إذا تعارضت مع وثيقة حاكمة، تسجل الملاحظة ويطبق المصدر الأعلى فورًا.

## 13. خلاصة التوجيه

> **ابنوا ASIE للسعودية أولًا، ببيانات سعودية وأدلة قابلة للتتبع، وذكاء اصطناعي عربي محكوم لا يملك الأرقام أو القرار. أكملوا رحلة المستخدم والقرار قبل توسيع النطاق أو الوثائق أو القدرات المستقبلية.**
